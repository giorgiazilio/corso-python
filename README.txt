
# AntennalMag Tracking & Analysis Pipeline

Welcome to the **AntennalMag** repository. This project provides an end-to-end data science and computer vision pipeline designed to analyze honeybee antennal movements from video recordings tracked via DeepLabCut (DLC). 

The repository includes tools for global configuration, manual/automated data cleaning, visual spike inspection, and downstream behavioral machine learning (Clustering and Classification).

---

## 1. Project Architecture & File Interactions

Understanding how the files interact is crucial before launching the pipeline:

                  [ config.py ]  <-- (Centralizes Paths & Thresholds)
                       |
         +-------------+-------------+
         |                           |
         v                           v
 [ spike_inspector.py ]     [ AntennalMag_dorsal.ipynb ]
 (Standalone GUI Player)    (Main ML & Analytics Pipeline)
         |                           |
         v                           v
  Extracts & Saves           Generates Plots &
  *_bad_frames.csv           Statistical Tables (CSVs)

### Component Overview
* config.py: The single source of truth. Every other script or notebook imports this file to read paths, Frame Rate (FPS), velocity thresholds, and body part naming conventions. Changing a parameter here updates it everywhere.
* spike_inspector.py: A standalone interactive computer vision tool. It scans raw tracking data for biologically impossible velocity anomalies ("spikes"), lets you visually inspect those exact video frames, and allows you to manually tag bad tracking sequences.
* Data Directory Output: Any frames flagged in the inspector are saved as [Video_Name]_bad_frames.csv into a dedicated subfolder defined by BAD_FRAMES_DIR in config.py.
* AntennalMag_dorsal.ipynb: The central Jupyter Notebook where the scientific analysis happens. It processes coordinates, calculates metrics using sliding windows, filters noise (optionally reading the manual bad frames generated by the inspector), runs a Principal Component Analysis (PCA) + K-Means clustering, and trains a Random Forest Classifier.

## 2. Environment Setup (Recommended)

To guarantee that all dependencies interact correctly without breaking existing software on your machine, it is highly recommended to isolate this project inside a dedicated Conda environment.

### Step 1: Create the Environment
Open your terminal (or Anaconda Prompt) and create a clean environment using Python 3.11:
conda create -n antennal_mag python=3.11 -y

### Step 2: Activate the Environment
conda activate antennal_mag

### Step 3: Install Required Packages
Run the following commands to install the exact scientific, machine learning, and computer vision libraries needed for the joint execution of all files:

# Core data science and numerical libraries
pip install numpy pandas scipy

# Machine learning, preprocessing, and statistical modeling
pip install scikit-learn

# Visualization tools and Jupyter environment
pip install matplotlib jupyter

# Computer Vision tools (Required for the interactive player)
pip install opencv-python

## 3. Step-by-Step Workflow Guide

To run the full analysis correctly, follow this sequence:

### Phase 1: Configuration
1. Open config.py in a text editor.
2. Edit the FOLDER path variable to point to the directory where your high-speed videos and DeepLabCut .csv tracking files are stored.
3. Adjust biological constants (e.g., FPS or VELOCITY_THRESHOLD) if your experimental setup changes.

### Phase 2: Visual Inspection (Standalone Tool)
Run the spike inspector from your terminal to clean tracking artifacts:

Option A: Run using the default video specified in config.py
python spike_inspector.py

Option B: Run specifying a specific video file path
python spike_inspector.py /path/to/your/specific_video.mp4

#### Keyboard Shortcuts inside the GUI:
* N or -> (Right Arrow)  : Jump to the next suspicious velocity spike frame
* P or <- (Left Arrow)   : Jump to the previous suspicious velocity spike frame
* . (Period)             : Fine-tune navigation: Advance forward by exactly 1 frame
* , (Comma)              : Fine-tune navigation: Step backward by exactly 1 frame
* M                      : Toggle Mark/Unmark: Flags current frame as "bad tracking" (Red Border)
* Q or ESC               : Save and Quit: Safely exits player and exports results to a CSV log

### Phase 3: Global Analytics & Machine Learning
1. Launch Jupyter Notebook by typing "jupyter notebook" or "jupyer-Lab" in your terminal.
2. Open AntennalMag_dorsal.ipynb.
3. In the Data Cleaning Setup cell, configure your artifact strategy via the configuration flags:
   - Set USE_MANUAL_SPIKES = True if you want the pipeline to read the clean-up logs exported by the spike_inspector.py.
   - Set AUTO_REMOVE_SPIKES = True to let the algorithm drop noisy frames using standard statistical thresholds.
4. Run all cells sequentially to generate behavioral sliding-window metrics, PCA data clustering matrices, and validation reports.

## 4. Pipeline Outputs

When running the full code, the pipeline automatically populates your workspace with the following assets:

* descriptive_statistics_condition.csv: A clean, single-level tabular matrix containing aggregated statistics (median, mean, standard deviation, count) for all antennal kinematics, optimized for direct import into Microsoft Excel.
* descriptive_by_condition.pdf: Multi-panel exploratory data plots tracking every single video recording trajectory alongside group medians and interquartile ranges (IQR).
* features_windows.csv: The segmented matrix splitting continuous time-series tracks into independent sliding windows, used as the primary tabular asset for Machine Learning operations.
